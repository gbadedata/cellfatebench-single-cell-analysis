# Evidence Map

## Project

CellFateBench: A Single-Cell Benchmark for Trajectory, Spatial, and Topological Analysis

Repository:

https://github.com/gbadedata/cellfatebench-single-cell-analysis

## Purpose of This Evidence Map

This document maps the major project claims to the files, outputs, tests, and workflow evidence that support them.

CellFateBench is designed as a benchmark-design project for evaluating reasoning over single-cell genomics workflows. It focuses on three benchmark areas:

1. trajectory and pseudotime reasoning;
2. spatial gene-pattern reasoning;
3. topological persistence reasoning.

The project includes controlled synthetic data, public task files, hidden answer keys, oracle outputs, deterministic validators, partial-credit scoring, calibration logs, Docker reproducibility, GitHub Actions CI, and reviewer-friendly outputs.

## Dataset Evidence

The controlled synthetic dataset is generated by:

* `src/cellfatebench/synthetic.py`
* `scripts/01_generate_synthetic_dataset.py`

Generated outputs:

* `data/synthetic/synthetic_cell_metadata.csv`
* `data/synthetic/synthetic_expression_matrix.csv`
* `data/synthetic/synthetic_gene_metadata.csv`
* `data/synthetic/synthetic_hidden_truth.json`

The dataset contains:

* 900 synthetic cells;
* 60 genes;
* known root/progenitor state;
* known transition state;
* two known terminal branches;
* known pseudotime values;
* known spatial coordinates;
* known spatial domains;
* known spatially variable gene groups;
* known topological design.

This controlled design is intentional. Benchmark tasks require known hidden truth so that solver answers can be evaluated deterministically.

## Hidden Answer Separation Evidence

Public benchmark tasks are stored separately from hidden answer keys.

Public directory:

* `benchmark_tasks/public/`

Hidden answer directory:

* `benchmark_tasks/hidden/`

This separation prevents answer leakage and supports benchmark-style evaluation.

The test suite includes checks that public task files do not expose hidden answer-key fields.

Relevant tests:

* `tests/test_trajectory_tasks.py`
* `tests/test_spatial_tasks.py`
* `tests/test_topology_tasks.py`

## Topology Evidence

Topology-aware summaries are generated by:

* `src/cellfatebench/topology.py`
* `scripts/04_generate_topology_summary.py`

Output:

* `results/tables/topology_summary.json`

The topology layer uses GUDHI to compute lightweight Rips persistence summaries from coordinate-derived point clouds.

The topology tasks are designed to evaluate whether a solver can distinguish:

* bifurcating cell-state structure;
* spatial ring-pattern evidence;
* false-positive claims that confuse spatial topology with cell-fate trajectory topology.
